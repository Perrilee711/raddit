# 项目交付模板目录

这个目录是给下一个项目直接复制使用的空白骨架。

## 使用方式

1. 复制整个 `project-delivery-template` 目录到新项目
2. 把 `{{PROJECT_NAME}}`、`{{DATE}}`、`{{OWNER}}` 这些占位符替换掉
3. 先写 `BRD`
4. 再写 `PRD`
5. 然后补 `UI / Technical / QA / Release`

## 目录结构

- `docs/delivery/`：正式交付文档
- `docs/product/`：产品原型与补充方案
- `docs/reports/`：研究结果、周报、专项分析
- `deploy/`：部署、运行与运维配置

## 模板目标

不是只复用文档名，而是复用完整闭环：

- 业务定义
- 产品定义
- 设计定义
- 技术定义
- 测试验收
- 上线交接
- 复盘复用
