## 🚨 异常速查表

| 你看到 | 大概率什么坏了 | 怎么办 |
|---|---|---|
| 看板进不去 | 密码错了 | 查 1Password 最新密码 |
| bridge health 没响应 | 服务挂了 | 找 Claude，或 SSH 上服务器 `sudo systemctl status fishgoo-mcp-bridge` |
| 看板能进但数字是昨天的 | 自动刷新挂了 | 找 Claude，或 SSH 跑 `sudo systemctl start fishgoo-daily-refresh.service` |
| GitHub 很久没新 commit | 大概率正常 | 没活干就不会有 commit |

## 🎯 进阶 · 专业版 1 分钟自检（需要终端）

如果你愿意跑 terminal 命令（5 条 curl + SSH），可以看完整的服务器状态：

```bash
# 1. bridge health
curl -s https://mcp.perrilee.com/fishgoo-bridge/health

# 2. 看板（有密码）
curl -s -u "perri:$(op read 'op://Private/Fishgoo Ad Board Basic Auth/password')" \
     https://mcp.perrilee.com/ad-board/ | head -3

# 3. 自动刷新 timer 下次触发
ssh tencent-chinawholesalebuy "systemctl list-timers | grep fishgoo"

# 4. 看板 HTML 最近一次 mtime
ssh tencent-chinawholesalebuy "stat -c 'mtime: %y' /var/www/fishgoo-ad-board/index.html"

# 5. repo 同步状态
cd /Users/perrilee/Desktop/探索/raddit && git fetch -q && \
  [ "$(git rev-parse HEAD)" = "$(git rev-parse origin/main)" ] && \
  echo "✅ synced" || echo "⚠️ out of sync"
```

（上面第 2 条用了 1Password CLI `op`，需要先 `brew install 1password-cli` + 登录。如果没装，直接在 URL 手输密码也可以。）
