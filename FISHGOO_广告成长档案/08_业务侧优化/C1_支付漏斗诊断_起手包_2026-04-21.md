**日期**：2026-04-21（Day 27 晚 Perri 批准启动）

**目标**：**把下单→支付转化率从 14.8% 提到 25%+**（等效 ROAS 从 3.41 → 5.77 · +69%）
**时间预算**：本周 Perri 每天 30 分钟 + 一次 1 小时和运营同事对齐

## 🎯 为什么做这件事（先统一判断）

**合伙人视角的结论**：

```
同样 $538 广告花费

  14.8% 下单→支付 → ROAS 3.41   ← 现状
  25.0% 下单→支付 → ROAS 5.77   ← +69% · 甩任何 Ads 优化
  35.0% 下单→支付 → ROAS 8.08   ← +137%
```

过去 28 天我们把精力都花在"广告端"。但真正被锁死的是**业务端的转化漏斗**。**Ads 端再怎么调，天花板就是 ROAS 3.5**。业务端动一下，天花板直接抬到 5.7+。

代购业务的 14.8% 下单→支付转化率是偏低的（行业 20-30%）· 意味着每 10 个下单用户里有 8 个没付钱就走了。这里面一定有可修的东西。

**我的判断**：这是你 5 月唯一必做的事。其他都可以推。

## 📋 本周要拿到的 3 组数据（由 Perri + 运营 + 技术同事协作）

### 数据组 A：漏斗节点明细（业务后台拉 / 技术同事出数）

我们要看的是**下单后到支付成功之间**每一步的流失率：

| 漏斗节点 | 当前已知数据 | 需要补的数据 |
|---|---|---|
| 访问商品页 UV | 未知 | 从 GA4 或业务后台拉 |
| 加入购物车 | 未知 | 业务后台 |
| 进入结算页 | 未知 | 业务后台 |
| **提交下单**（未付款） | 4/8-4/20 共约 240+ 单（估算） | 业务后台按天拉 |
| **支付成功** | 36 单（Metabase SEM 口径） | 已知 |
| **下单→支付率** | **14.8%** | 已知 |

**要从业务后台拉的 Metabase card / report**：
- "下单未支付"用户按日统计（至少 30 天窗口）
- 下单流失用户的特征（常用：是新用户还是老、平均订单金额、地址国家、支付方式选了啥再取消）
- 支付方式分布（信用卡 / 支付宝 / 微信 / PayPal 各多少）

### 数据组 B：流失归因（问运营同事）

让运营同事凭经验回答 5 个问题：

1. **"最近 30 天，下单未支付的用户，客服接到过他们的问题吗？问的最多的是什么？"**
   - 可能答案：物流太贵 / 运费没写清楚 / 商品缺货 / 支付方式不够 / 信任感差
2. **"你们遇到过用户下单后等了很久才支付（>24h）吗？他们说什么原因？"**
3. **"代购业务下单和支付之间有没有'代购报价'的环节？如果有，报价偏贵用户会流失吗？比例多少？"**
4. **"最近 30 天，我们主动联系过下单未支付的用户吗？如果联系过，挽回率是多少？"**
5. **"你觉得 14.8% 这个数字是哪一步在卡？你的直觉是哪一步最宽？"**

**这 5 题不要一次性问完 · 本周分 2-3 次问 · 每次聊 15 分钟**。运营的判断直觉是比数据更快的线索。

### 数据组 C：自己体验一次下单（最重要 · Perri 亲自做）

**本周内 Perri 自己走一次 fishgoo.com 下单到支付的完整流程**（别让同事代跑，亲自用手机 + 电脑各跑一次）：

- 找 3 个目标代购商品（低价 $30 + 中价 $100 + 高价 $300 各一个）
- 从首页进 → 加购 → 结算 → 填地址 → 选支付 → **停在支付确认那一刻，不付**
- 记录：
  - 每一步用了多久
  - 哪一步出现了"我犹豫"的瞬间（自问：我为什么会想放弃？）
  - 地址字段合不合理（代购业务地址复杂，中转仓地址好填吗？）
  - 物流费是什么时候出现的？让人惊喜还是肉痛？
  - 支付方式选择页让你想选哪个？选错了能切吗？
  - 看没看到"预计到货时间"？有没有让你安心？

**这个动作是最关键的**。所有数据的意义，都不如你自己走一次流程。

## 🧠 我预判的 5 个漏斗瓶颈（仅供参考 · 你自己体验后再校正）

基于行业经验 + 代购业务特性，我猜 14.8% 这个数字卡在：

1. **物流费不透明 / 出现太晚** → 用户下单后发现运费 $30-80 退缩（代购业务常见）
2. **代购报价环节拉长** → 下单后要等客服回复精确价格，用户流失窗口被拉长 12-48h
3. **支付方式不够 / 切换麻烦** → 信用卡失败切支付宝要重下单，断片
4. **地址填写复杂** → 代购中转仓地址要填两次，用户填到一半放弃
5. **信任感不足** → 新用户下单后没看到"保障条款 / 退款政策 / 物流追踪预告"，付款前怀疑

**我的直觉最大可能是 1 + 2**（物流 + 报价）· 这两个如果属实，改起来不难（提前展示物流费 + 报价快速化）· 预期能把 14.8% → 22%。

**但我的直觉不是数据 · 数据要靠你本周拉回来**。
